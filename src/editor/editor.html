            Convert — importing files
            
                
              
                Upload a file on the Convert tab by dropping it on the upload zone or clicking to browse. The format is detected from the file extension:
                
              
                  
                
                • Spreadsheets (.csv, .xlsx, .xls) — row 1 must contain MARC notation headers such as 245//$a (title), 1001/$a (author), or 650/0$a (subject). Each following row becomes one record. Columns without MARC notation open the Column mapping panel so you can assign notation and reorder columns before import.
                  • 
                
                • MARC files (.mrc, .mrk, .xml, .txt) — opened directly. The record type (bibliographic, authority, holdings) is detected from the Leader, and fixed fields are padded to their correct MARC21 lengths.
                  • 
              
                
              
                After a successful import you are taken to the Edit tab automatically. A "Download example spreadsheet" link on the Convert tab shows the expected header layout.
                
              
                Save session downloads a .marclite.json file with all records and settings. Load session restores from that file. Restore draft reloads the auto-saved browser draft — if your current session differs, you can discard current records, merge with the draft, or overwrite completely.
                
            
                
          
                

          
                
            Edit — records and fields
            
                
              
                The Edit tab has a toolbar for creating records and a two-panel workspace:
                
              
                  
                
                • New record toolbar — choose a Record type and click New record for a minimal blank record. After you Save as template from the field editor, saved setups appear in the Template dropdown (filtered by record type) so you can create new records from them.
                  • 
                
                • Records list (left) — click a record to open it. Use Filter, Search (title, author, tag text), and Validation profile to narrow the list. Duplicate copies scoped/selected records; an Undo duplicate option appears after duplicating. Delete removes records after confirmation. Checkboxes (with Select all / Clear selection) define scope for Batch, Cleanup, and Export.
                  • 
                
                • Field editor (right) — edit the open record in Form or Mnemonic view. Save as template stores the current leader and field layout (001/005 cleared) for reuse when creating records. Edits save automatically; Save record normalizes and re-validates. Undo reverts the last field edit, batch, or cleanup. Paste field inserts a copied field. Previous/Next step through the filtered list. Add MARC field opens a dialog for data or control fields. Each field card has Remove; data fields also have Add subfield, reorder, duplicate, and copy actions.
                  • 
              
                
              
                The MARC21 preview below the editor updates live as you type.
                
            
                
          
                

          
                
            Order — rearranging records
            
                
              
                The Order tab lets you change the sequence of loaded records after import. Use Move up / Move down, or enter a target position and click Move. Record numbers shown throughout MARCLite (batch scope, export, compare) follow this order.
                
            
                
          
                

          
                
            Leader & 008 — segmented fixed fields
            
                
              
                The Leader (24 characters) and field 008 (40 characters for bibliographic and authority records, 32 for holdings) are fixed-position fields. MARCLite splits them into labelled segment inputs matching the Library of Congress MARC21 positions — for example the 008 publication year sits in positions 07–10 and language in 35–37.
                
              
                  
                
                • Hover a segment to see a tooltip explaining its codes.
                  • 
                
                • The combined value is shown beneath the segments and updates as you type.
                  • 
                
                • A link under each editor opens the relevant LoC documentation page.
                  • 
                
                • Short values are padded with spaces automatically; imported fields are aligned to the correct positions on load.
                  • 
              
                
              
                Other control fields (001 record identifier, 005 timestamp) are free-form and use a single value input.
                
            
                
          
                

          
                
            Indicators & field tags
            
                
              
                Each data field card shows a Tag input (three digits) and an Indicators (2 chars) input. Enter indicators together, e.g. 10 or 00.
                
              
                Common example — the 245 title field:
                
              
                  
                
                • First character: 1 when the record has a main author entry (1XX field), 0 when it does not.
                  • 
                
                • Second character: number of characters to skip when sorting — e.g. 4 for a title starting with "The ".
                  • 
              
                
              
                Edits save automatically as you type; use Save record to normalize and re-validate the open record. You can also change tags or indicators across many records from the Batch tab.
                
            
                
          
                

          
                
            Validation — finding and fixing issues
            
                
              
                Records are validated continuously — on import, creation, and every edit. Choose a Validation profile on the Edit tab (Cataloguing, Strict, or Spreadsheet import). Results appear in the banner beneath the navigation tabs:
                
              
                  
                
                • Red issues are errors (structurally invalid MARC); amber issues are warnings (unusual but legal).
                  • 
                
                • Click the banner to expand the full list. Issues affecting several records are grouped with a record range.
                  • 
                
                • Every issue includes a "How to fix" hint explaining where in the tool to correct it.
                  • 
                
                • Click an issue to jump directly to the affected record and field, which is highlighted and focused.
                  • 
                
                • Some issues offer a one-click Fix button in the banner.
                  • 
                
                • Grouped issues offer a Batch edit button that pre-fills the Batch tab with the right scope and targets.
                  • 
              
                
              
                Invalid inputs are outlined in red (errors) or amber (warnings) inside the field editor as you type.
                
            
                
          
                

          
                
            Batch — bulk find & replace
            
                
              
                The Batch tab has two areas: Batch presets at the top for saving and loading common find/replace setups, and Batch editing below for running operations.
                
              
                  
                
                • Record scope — all records, a custom range (e.g. 1-75 or 1, 6, 17), or the current record. You can also tick record checkboxes on the Edit tab.
                  • 
                
                • Record type — optionally limit batch operations to bibliographic, authority, or holdings records within the scope above.
                  • 
                
                • Find/Replace — matched literally by default; patterns like \d+ or /pattern/ are detected as regular expressions automatically.
                  • 
                
                • Tag and subfield filters — restrict find/replace, e.g. tag 650 subfield a. Leave Find empty with both set to replace the entire subfield value (use Replace as the new value).
                  • 
                
                • Update targets — choose what to touch: leader, control values/tags, indicators, subfield codes/values.
                  • 
                
                • Preview changes shows the diff before applying; Undo all reverts an applied batch.
                  • 
              
                
              
                Normalize all records tidies common issues, and Delete tag from all strips a field from every scoped record.
                
            
                
          
                

          
                
            Cleanup — standardising data
            
                
              
                The Cleanup tab fixes common metadata problems in the scoped records. Use the same Record scope and Record type controls as Batch to limit which records are affected. Toggle which fixes run:
                
              
                  
                
                • Remove duplicate fields — identical repeated fields are dropped.
                  • 
                
                • Standardize punctuation — trailing punctuation follows MARC conventions.
                  • 
                
                • Fix encoding — repairs common character-encoding damage.
                  • 
                
                • Normalize ISBNs — strips hyphens and stray text from 020 fields.
                  • 
                
                • Normalize dates — standardises date formats.
                  • 
              
                
              
                Use Preview cleanup to see the changes first; Undo all reverts an applied cleanup.
                
            
                
          
                

          
                
            Export — saving your records
            
                
              
                The Export tab has three control areas:
                
              
                  
                
                • Record scope — specify record numbers (e.g. 1-75) and optionally filter by record type, same as Batch/Cleanup.
                  • 
                
                • Export range — export all loaded records, the scoped set above, or only records visible in the Edit list.
                  • 
                
                • Export options — pick a format, optionally block export on validation errors, export, or check 856 $u URL links within the chosen range.
                  • 
              
                
              
                Supported formats:
                
              
                  
                
                • MARC mnemonic (.mrk) — human-readable text, compatible with MarcEdit.
                  • 
                
                • MARC binary (.mrc) — the standard ILS interchange format.
                  • 
                
                • MARCXML (.xml) — the LoC XML schema.
                  • 
                
                • CSV / JSON — delimited or structured data for spreadsheets and scripts.
                  • 
                
                • Dublin Core / MODS / BIBFRAME — converted metadata formats.
                  • 
              
                
              
                The Export preview shows a live sample of the output. Tick "Block export on validation errors" to prevent exporting records with structural problems. Check URL links uses the same export range and inspects http(s) addresses in MARC field 856 subfield $u.
                
            
                
          
                

          
                
            Compare — side-by-side records
            
                
              
                The Compare tab shows two records as MARC mnemonic text in side-by-side panels. Choose records from the dropdowns (shown as number — title) and click Compare, or use Use current record as A to start from the record open in the Edit tab.
                
            
                
          
                

          
                
            Sessions & drafts — saving your work
            
                
              
                MARCLite auto-saves a draft to browser storage as you work (about every 1.5 seconds). On the Convert tab:
                
              
                  
                
                • Save session — download a portable .marclite.json file with records, scope, validation profile, and column mapping settings.
                  • 
                
                • Load session — restore from a saved session file.
                  • 
                
                • Restore draft — reload the auto-saved draft. If your current session differs, choose to delete new records and restore the draft, merge draft with current records, overwrite completely, or cancel.
                  • 
                
                • Clear draft — remove the auto-saved draft from browser storage.
                  • 
              
                
              
                If you create a new record and the draft no longer matches what you expect, the auto-save may have updated the stored draft — use Save session regularly for a backup you control.
